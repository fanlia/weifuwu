# Org — 页面布局方案

> 当前布局的问题与改进方向。
> 核心目标：不同页面类型用不同布局，让信息层级更清晰。

---

## 一、当前布局的问题

### 1.1 所有页面共用同一布局

```
┌─────────────┬────────────────────────┐
│  侧栏       │   RouteView            │
│  (260px)    │                        │
│             │  所有页面共用这个区域    │
│  OrgTree    │                        │
│             │  - 首页（租户列表）     │
│             │  - 公司列表            │
│             │  - 部门列表            │
│             │  - 部门聊天            │
│             │                        │
└─────────────┴────────────────────────┘
```

问题是：**聊天页和浏览页用同一个布局**。聊天需要最大化消息区域，浏览需要清晰的信息展示。

### 1.2 聊天页布局拥挤

```
DepartmentPage 内部：
┌──────────────────────────────────────────┐
│  ← 返回公司  AI Team    成员(2) 📚知识库  │  ← header 一排按钮
├─────────────────────┬────────────────────┤
│                     │  成员面板 (w-72)    │  ← 和 KB 共用侧栏
│     聊天区域         │  👤 张三           │
│                     │  🤖 智能助手       │
│                     │                    │
│                     │  + 添加            │
├─────────────────────┴────────────────────┤
│  @ 🤖 输入消息...                   [发送] │
└──────────────────────────────────────────┘
```

问题：
- 成员面板和 KB 面板**共用一个位置**，不能同时看
- 聊天区域宽度被挤压
- 边栏只有 288px，放 KB 表单太窄

### 1.3 缺少会话列表

Slack 和 Discord 都有一个**中间栏**展示当前用户的会话/频道列表，Org 目前只有侧栏的树形结构，缺少"我最近在聊什么"的快捷入口。

## 二、布局方案

### 2.1 三种布局模式

根据页面类型，自动切换布局：

```
浏览模式 (Home/Tenant/Company)：
┌─────────────┬──────────────────────────────┐
│  侧栏       │  内容区                       │
│  (260px)    │  卡片列表 / 表格              │
│  组织树     │                              │
└─────────────┴──────────────────────────────┘

聊天模式 (Department)：
┌─────────────┬──────────────┬───────────────┐
│  侧栏       │  会话列表     │  聊天 + 面板   │
│  (220px)    │  (240px)     │  (flex-1)     │
│  组织树     │  部门1 💬    │               │
│             │  部门2 💬    │  ← 正在聊天    │
│             │  部门3 💬    │               │
│             │              │  面板可切换     │
│             │              │  成员/KB/文件   │
└─────────────┴──────────────┴───────────────┘

全屏模式 (登录/弹窗)：
┌────────────────────────────────────────────┐
│                                            │
│              ┌──────────────┐               │
│              │  登录卡片     │               │
│              │              │               │
│              └──────────────┘               │
│                                            │
└────────────────────────────────────────────┘
```

### 2.2 布局组件结构

```typescript
// 布局容器 — 根据当前路由选择布局
function AppLayout(props: {}, ctx: WfuiContext) {
  // 判断当前页面类型
  const isChat = computed(() => ctx.route.path.includes('/dept/'))
  
  return (
    <Show when={isChat} fallback={<BrowseLayout />}>
      <ChatLayout />
    </Show>
  )
}

// 浏览布局 — 侧栏 + 内容
function BrowseLayout() {
  return (
    <div class="flex h-screen">
      <Sidebar />
      <main class="flex-1 overflow-y-auto">
        <RouteView />
      </main>
    </div>
  )
}

// 聊天布局 — 侧栏 + 会话列表 + 聊天
function ChatLayout() {
  return (
    <div class="flex h-screen">
      <Sidebar />
      <ConversationList />
      <div class="flex-1 flex flex-col">
        <RouteView />
      </div>
    </div>
  )
}
```

### 2.5 聊天面板

```
┌──────────────────────────────────────┐
│  ← AI Team                 【👥  📚】│  ← header: 48px
├──────────────────────────────────────┤
│                                      │
│  ┌──────────────┐  ┌──────────────┐  │
│  │  消息区域      │  │  面板(tab)   │  │
│  │  flex: 1      │  │  w-72        │  │
│  │               │  │             │  │
│  │  消息列表      │  │  成员 | KB   │  │
│  │               │  │  文件 | 搜索  │  │
│  │               │  │             │  │
│  │               │  │             │  │
│  └──────────────┘  └──────────────┘  │
│                                      │
├──────────────────────────────────────┤
│  @ 🤖 输入消息...              [发送] │  ← input: 52px
└──────────────────────────────────────┘
```

面板使用 **Tab 切换**，支持同时查看成员和 KB：

| Tab | 内容 |
|-----|------|
| 👥 成员 | 部门 Agent 列表 + 添加 |
| 📚 知识库 | 文档列表 + 导入 + 搜索 |
| 📎 文件 | 共享文件（未来）|
| 🔍 搜索 | 消息全文搜索（未来）|

### 2.6 响应式行为

| 断点 | 侧栏 | 会话列表 | 右面板 |
|------|------|---------|--------|
| > 1200px | 显示 220px | 显示 240px | 显示 |
| 800-1200px | 图标 60px | 显示 200px | 隐藏，Tab 切换 |
| < 800px | 隐藏 Drawer | 全屏 | 隐藏 → Sheet |

## 三、布局切换逻辑

```
URL 路径 → 判断页面类型 → 选择布局

/                             → BrowseLayout
/tenant/:id                  → BrowseLayout
/tenant/:id/company/:id      → BrowseLayout
/tenant/:id/company/:id/dept/:id  → ChatLayout
其他                          → BrowseLayout
```

```typescript
function useLayoutType(ctx: WfuiContext): 'browse' | 'chat' {
  const path = ctx.route.path
  if (path.includes('/dept/')) return 'chat'
  return 'browse'
}
```
